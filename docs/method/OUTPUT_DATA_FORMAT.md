# 仿真项目输出数据形式说明

本文简要说明本仿真项目生成的数据长什么样、有哪些列、各列怎么用。

## 1. 输出文件

仿真运行后会输出一张时序数据表，通常保存为以下格式之一：

- `parquet`：推荐，体积小、读取快。
- `csv`：便于人工查看和临时联调。

每一行表示一个仿真时间步，默认步长为 `60 s`。表中一般会包含时间索引列、DCS 在线变量、过程化验变量和最终标签。

## 2. 数据类型

输出数据可分为四类：

1. `DCS` 在线变量  
   现场控制系统可直接观测到的量，例如流量、液位、压力、电流、阀位、泵频、泡沫高度等。

2. `lab_*` 过程化验变量  
   代表人工采样或实验室化验结果，只有在采样时刻有值，其余时刻通常为 `NaN`。

3. `y_fx_xin1` / `y_fx_xin2` 最终标签  
   两条浮选线的最终精矿品位标签，用于软测量建模与结果评估。

4. 隐藏状态 `_x_*`  
   这些是机理仿真的内部变量，例如解离度、粒度、中间物流状态等，默认不写入最终输出文件，只在仿真内部流转。

## 3. 列名约定

- `agg_*`：兼容旧 DCS 口径的聚合输出列。
- `MC1_*`：现场设备或仪表编号风格的 DCS 列。
- `fx_*`：浮选系统相关输出列。
- `lab_*`：过程化验列。
- `y_*`：目标标签列。
- `_x_*`：内部隐藏状态列。

## 4. 单位约定

项目中常见单位约定如下：

- 品位类：`0.67` 表示 `67%`，也就是分数形式。
- 化验类 `lab_*`：通常以百分数输出，例如 `67.1` 表示 `67.1%`。
- 流量类：通常为 `m3/h` 或等价体积流量口径，具体以列定义为准。
- 液位类：通常为 `m` 或归一化液位。
- 电流类：通常为 `A`。
- 泵频、阀位类：通常为 `Hz` 或 `0-1` 归一化值。

## 5. 推荐使用方式

- 训练在线软测量模型时，优先使用 `DCS` 在线变量作为输入。
- 做机理分析或特征泄漏检查时，可同时保留 `lab_*` 和 `y_*`。
- 不建议把 `_x_*` 作为模型特征，它们属于隐藏状态，不应直接进入训练集。

## 6. 一个输出样例

```text
timestamp, agg_tm_motor_current, agg_tm_cyclone_feed_flow, fx_s1_level,
lab_tm_overflow_f325, lab_flo_conc_tfe_s1, y_fx_xin1
2026-05-14 08:00:00, 312.8, 184.5, 0.63, NaN, NaN, 0.6712
2026-05-14 08:01:00, 313.1, 183.9, 0.64, NaN, NaN, 0.6711
2026-05-14 08:02:00, 313.4, 184.1, 0.65, 45.2, 67.4, 0.6710
```

这表示：在线 DCS 变量每步连续输出，化验值按采样时刻间歇出现，最终标签始终可用。
